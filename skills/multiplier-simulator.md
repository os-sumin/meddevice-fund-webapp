# 펀드 승수효과·조성규모 시뮬레이터

의료기기 정책펀드의 **타당성** 파트용 계산 모델. 정부 출자액 대비 민간 매칭·승수(레버리지),
연차별 조성 스케줄, 데스밸리 2구간 자금배분을 시나리오별로 산출한다. **웹서치 없는 결정적 계산**이므로
빠르고, Vercel로 옮길 때도 타임아웃 걱정 없는 순수 함수다.

## 정의 (고정)

- 민간출자 = 정부출자 × `private_multiple`
- 총 결성규모 = 정부출자 + 민간출자
- **승수(레버리지) = 총 결성규모 ÷ 정부출자 = 1 + private_multiple**
- 우선손실충당(first-loss) = 정부출자 × `first_loss_pct` (민간 다운사이드 방어분)
- 민간 커버비율 = 민간출자 ÷ first-loss (정부 우선손실 1원당 유치한 민간자금)

## 워크플로우

### 1단계 — 파라미터 확정
사용자에게서 또는 합리적 기본값으로 다음을 정한다:
- 정부 총 출자액(억원), 조성 연차(보통 3~5), 연차별 정부출자 배분 가중치(합=1, 생략 시 균등)
- 시나리오별 민간 매칭 배수(private_multiple)와 first-loss 비율. 최소 3개(보수/기본/공격) 권장.
- 데스밸리 2구간 배분(임상 : 초기사업화, 합=1)
파라미터가 안 주어지면 합리적 기본값으로 돌리고 "가정값" 명시 후 사용자가 조정하게 한다.

### 2단계 — 계산 + 출력
config JSON을 만들어 `scripts/simulate.py`로 계산·Excel 생성:

```bash
cd <스킬경로>
python scripts/simulate.py --config /tmp/sim_config.json --out /mnt/user-data/outputs/펀드_승수시뮬레이션.xlsx
```

config 스키마는 `scripts/simulate.py` docstring 참조(base: gov_contribution·years·annual_gov_weights·
death_valley_split / scenarios: name·private_multiple·first_loss_pct). 스크립트는 콘솔 요약도 출력한다.

Excel은 3개 시트: ①시나리오 요약(승수·first-loss·민간커버) ②연차별 스케줄(정부·민간·누적) ③데스밸리 배분.
생성 후 `present_files`로 제시하고, 채팅에는 시나리오 요약표 + 해석 2~3줄을 함께 낸다.

### 3단계 — 해석
숫자만 내지 말고 타당성 관점 해석을 붙인다: 어느 시나리오가 정책 목표 승수를 충족하는지, first-loss를 키우면
민간 커버가 어떻게 늘어나는지(리스크분담 ↔ 민간 crowd-in 트레이드오프), 3~5개년 중기 예산계획으로 얼마가 언제 필요한지.

## 출력 형식 (채팅 표 템플릿)

```
## 승수효과 시뮬레이션 (정부출자 <X>억원 기준, 단위: 억원)

| 시나리오 | 정부 | 민간 | 총 결성 | 승수 | first-loss | 민간커버 |
|---|---|---|---|---|---|---|
| 보수(1.5x) | 1,000 | 1,500 | 2,500 | 2.5배 | 200 | 7.5배 |
...

해석: <목표 승수 충족 여부 / first-loss 트레이드오프 / 연차별 필요재원>
```

## 주의
- 이건 계산 모델이지 시장·사례 데이터가 아니다. 매칭배수·first-loss의 "현실적 범위"는 meddevice-fund-benchmark의
  벤치마크 펀드 실제값에서 가져와 시나리오로 넣는다(두 스킬 연계).
- 억원 단위 고정. 사용자가 다른 단위를 쓰면 변환해서 일관되게.
- 가정값으로 돌렸으면 반드시 "가정" 명시. 실제 정책 목표 승수·예산은 진흥원과 협의 대상.
