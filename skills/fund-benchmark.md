# 의료기기 정책펀드 벤치마크 사례조사

KHIDI 의료기기 전용 정책펀드 조성방안 기획 연구용역을 지원하는 사례조사 스킬. 국내외 정책펀드를
**고정 비교 템플릿**으로 수집·정리해 비교표 + Excel로 낸다. 이 용역의 핵심은 산업분석이 아니라
**펀드 운용 거버넌스 설계**이므로, 사례조사는 각 펀드의 "거버넌스 구조를 어떻게 짰나"에 초점을 둔다.

## 핵심 원칙

- **바뀌는 수치는 반드시 재확인**. 결성규모·집행액·목표액·GP 명단은 시간이 지나면 변한다. `references/benchmark-funds.md`의
  참고값은 콜드스타트 방지용 시드일 뿐, 표에 넣기 전 `web_search`/`web_fetch`로 최신값을 확인하고 출처 URL을 남긴다.
- **의료기기 관점 고정**. 모든 사례는 "이 구조가 의료기기 펀드에 적용 가능한가/왜 안 맞나"의 시사점 컬럼으로 귀결시킨다.
  (예: K-바이오·백신 = 제약 중심·임상 10년+·대형 티켓 → 의료기기 사업화 3~5년·소액 다건·M&A 회수와 안 맞음 = 전용펀드 당위성)
- **거버넌스 유형으로 분류**. 각 펀드를 위탁형 / 그립강화형 / 직접운용형 중 하나로 분류해 본 용역 3안(2안=그립강화형이 추천안)에
  매핑한다. 이게 ④펀드모델 설계의 직접 인풋이 된다.
- **펀드 층위 구분 주의**: 자금원(부처) → 모펀드 운용기관(KVIC/성장금융) → 자펀드 GP(민간 VC) → 투자기업. 진흥원은 정책층엔
  관여해도 GP 자리엔 못 앉는다. "투자연계형 R&D"는 펀드가 아니라 R&D 지원 모델이므로 펀드 사례와 섞지 않는다.
- 결과는 정부·기관 보고서 톤으로 간결·정확하게. 확인 못 한 수치는 지어내지 말고 "확인 필요"로 남긴다.

## 워크플로우

### 1단계 — 대상 펀드 확정
요청에서 조사 대상을 정한다. "국내 다", "해외 다", "전체", 특정 펀드명 지정 등. 지정이 없으면 `references/benchmark-funds.md`의
시드 전체(국내 4~6 + 해외 4)를 기본 대상으로 한다. 시드에 없는 펀드를 요청하면 새로 조사해 추가한다.

### 2단계 — 시드 로드
`references/benchmark-funds.md`를 읽어 각 펀드의 "왜 벤치마크인가", 거버넌스 유형, 참고 수치를 확보한다.

### 3단계 — 최신값 검증 (필수)
각 펀드에 대해 다음을 `web_search`로 확인한다:
- 결성·목표 규모, 누적 집행액, 결성 연도/차수
- 재원구조(정부:민간 비율), 리스크분담 장치(first-loss·우선손실충당·성과우선배분 유무)
- 주목적 투자의무(섹터·비율·투자단계)
- 최신 GP 명단·회수 실적(있으면)
검색 예: `K-바이오·백신 펀드 결성 규모 2026`, `EIC Fund blended finance medtech size`, `국민성장펀드 우선손실 세제혜택`.
확인된 사실만 표에 넣고, 각 행에 출처 URL을 붙인다.

### 4단계 — 승수효과 산출
가능한 펀드는 승수효과 = 총 결성규모 ÷ 정부 출자액(레버리지 배수)을 계산해 넣는다. 정부 출자액이 불명확하면 "확인 필요".

### 5단계 — 출력 (비교표 + Excel)

**(a) 채팅 비교표**: 아래 템플릿의 마크다운 표. 표 아래에 "거버넌스 유형별 요약 3줄"(위탁형/그립강화형/직접운용형 각각
대표 사례와 본 용역 시사점)을 붙인다.

**(b) Excel**: 행 데이터를 JSON으로 만들어 `scripts/build_workbook.py`로 .xlsx 생성.

```bash
cd <스킬경로>
python scripts/build_workbook.py --rows /tmp/fund_rows.json --out /mnt/user-data/outputs/정책펀드_벤치마크.xlsx
```

JSON 스키마는 `scripts/build_workbook.py` 상단 docstring 참조(fund, region, authority, year, size,
funding_structure, risk_sharing, governance, gov_type, mandate, term_exit, multiplier,
meddevice_implication, source). 생성 후 `present_files`로 제시한다.

## 주의

- 규모·집행액·GP 명단을 **시드값 그대로 쓰지 말 것**. 반드시 재확인(3단계).
- 의료기기 시사점 컬럼을 비우지 말 것 — 이 스킬의 존재 이유가 그 컬럼이다.
- 사용자가 특정 축(예: "리스크분담 구조만 비교")을 요청하면 해당 컬럼 중심으로 좁혀서 낸다.
- 시장규모·성장률 등 산업 데이터 요청은 이 스킬 범위 밖(②의료기기 시장조사 스킬 소관). 사례·구조·거버넌스에 집중한다.
